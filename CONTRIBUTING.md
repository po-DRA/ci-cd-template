# Contributing

Thank you for your interest in improving this teaching template!

## How to contribute

Contributions are welcome primarily through **GitHub Issues**. This keeps all
feedback visible, searchable, and easy to triage.

### Report a bug

Open an issue at **[Issues → New issue](https://github.com/po-DRA/ci-cd-template/issues/new)**
and include:

- Which flow or section the problem is in (e.g. "Flow 3 — Hypothesis")
- What you expected to happen
- What actually happened (paste the error output if applicable)
- Your operating system and whether you are running locally or in Codespaces

### Suggest an improvement

Open an issue with the label **enhancement** and describe:

- What is unclear, missing, or could be explained better
- The audience level you have in mind (beginner / intermediate)
- Any concrete wording or code you would suggest

### Request a new flow or topic

Open an issue with the label **feature request**. Include:

- The tool or concept you would like covered
- Why it fits this template's scope (hands-on ML workflow for teaching)

## What we are not looking for (at this stage)

- Large refactors of the source code or test suite without prior discussion
- Changes that increase the complexity of the core learning path
- New dependencies that are not widely used in the Python ML ecosystem

## Code of conduct

Be kind and constructive. This is a teaching resource used by learners at all
levels — all feedback should be framed accordingly.
